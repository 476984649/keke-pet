# 🐱 可可桌宠（Keke Desktop Pet）产品需求文档

> 版本：v1.0 | 日期：2026-07-07 | 作者：产品经理

---

## 一、市场调研

### 1.1 当前热门桌宠产品分析

| 产品 | 类型 | 核心玩法 | 传播力 | 付费模式 | 不足 |
|------|------|---------|--------|---------|------|
| **Desktop Goose** | 恶搞型 | 桌面鹅拖泥巴、抢鼠标、弹窗骚扰 | ⭐⭐⭐⭐⭐ 病毒式 | 买断$5 | 内容少、易腻 |
| **Shimeji** | 陪伴型 | 小人/角色在桌面走动、爬窗口、复制自己 | ⭐⭐⭐⭐ 开源生态 | 免费+角色包 | UI老旧、无养成 |
| **Bongo Cat** | 互动型 | 猫咪敲击桌面、跟随鼠标 | ⭐⭐⭐⭐ 表情包传播 | 免费 | 功能单一 |
| **eSheep** | 怀旧型 | 电子羊在桌面走动、睡觉 | ⭐⭐⭐ 情怀 | 免费 | 20年未更新 |
| **Desktop Mate** | 3D型 | 3D角色在桌面跳舞互动 | ⭐⭐⭐ VR联动 | 免费+DLC | 性能要求高 |
| **Virtual Cottage** | 氛围型 | 角色在桌面角落学习/工作 | ⭐⭐ 小众 | 免费 | 无互动 |
| **Nekopara桌宠** | 粉丝向 | 猫娘桌面互动、语音 | ⭐⭐⭐ 粉丝经济 | 买断 | 需IP支撑 |

### 1.2 市场空白与机会

```
        互动性高
           │
  Desktop Goose ●    │    ● Nekopara
           │
           │     ★ 可可桌宠定位
           │    （治愈陪伴 + 趣味互动 + 养成）
           │
  ──────────┼──────────→ 养成深度
           │
  Shimeji ● │    ● Virtual Cottage
           │
    Bongo Cat ●
           │
        互动性低
```

**核心洞察**：市面上缺少一款**既有深度养成、又有趣味互动、画风治愈**的桌宠。Desktop Goose 好玩但没养成，Shimeji 有生态但没成长感。

### 1.3 对标策略

- **行为系统** 对标 Shimeji（走动、爬窗、多实例）
- **互动趣味** 对标 Desktop Goose（惊喜事件、小恶作剧）
- **动画质感** 对标 Bongo Cat（帧动画流畅可爱）
- **养成深度** 自研（好感度、装扮、成就）

## 二、产品定位

### 2.1 一句话介绍

> 一只叫可可的蓝山双布偶猫住在你的电脑里，陪你工作、捣乱、长大。

### 2.2 目标用户

| 画像 | 特征 | 需求 |
|------|------|------|
| 学生/白领 | 长时间面对电脑 | 陪伴减压、摸鱼乐趣 |
| 猫奴 | 喜欢猫但无法养 | 云养猫替代满足 |
| 二次元用户 | 喜欢可爱角色 | 收集装扮、截图分享 |
| Steam玩家 | 轻度游戏习惯 | 后台挂机、成就驱动 |

### 2.3 可可猫设

| 属性 | 设定 |
|------|------|
| 品种 | 蓝山双布偶猫（Blue Bicolor Ragdoll） |
| 外貌 | 圆脸、蓝眼、面部倒V白色花纹、白四肢、蓝灰耳朵尾巴 |
| 性格 | 认真、温柔、偶尔犯困、喜欢被摸头 |
| 口头禅 | "喵~"（不同语调表达不同情绪） |

## 三、功能需求

### 3.1 核心功能（MVP）

#### F1. 桌面存在感
- 可可猫以透明窗口形式常驻桌面最上层
- 窗口大小约 150×200px（可缩放）
- 支持拖拽移动可可位置
- 启动时从屏幕边缘走进来

#### F2. 自主行为系统
可可根据**状态机**自主切换行为：

```
         ┌──────────┐
    ┌───→│  睡觉 💤  │←───┐
    │    └─────┬─────┘    │
    │          │ 醒了      │ 长时间
    │          ↓          │ 无操作
    │    ┌──────────┐    │
    │    │  待机 😺  │←───┘
    │    └─────┬─────┘
    │          │
    │     ┌────┼────┐
    │     ↓    ↓     ↓
    │  🚶走路  🪟爬窗  🎯盯鼠标
    │     │    │     │
    │     └────┼─────┘
    │          ↓
    └──── 回到待机
```

| 行为 | 触发条件 | 持续时间 | 动画 |
|------|---------|---------|------|
| 待机 | 默认状态 | 5-15秒 | 坐着眨眼、摇尾巴 |
| 走路 | 随机触发 | 3-8秒 | 从A点走到B点 |
| 爬窗口 | 靠近窗口边缘 | 直到掉下来 | 攀爬、悬挂、掉落 |
| 盯鼠标 | 鼠标靠近可可 | 持续跟随 | 头跟随鼠标转动 |
| 睡觉 | 5分钟无操作 | 直到唤醒 | 蜷缩、呼吸起伏 |
| 打哈欠 | 随机 | 2秒 | 张嘴、伸懒腰 |
| 舔毛 | 待机中随机 | 3秒 | 舔爪子、洗脸 |

#### F3. 鼠标互动

| 操作 | 行为 | 反馈 |
|------|------|------|
| 单击可可 | 摸头 | 可可眯眼、发出"喵~"、+1好感度 |
| 双击可可 | 弹跳惊吓 | 可可跳起来、落地后瞪你 |
| 拖拽可可 | 被拎起来 | 四肢垂挂、委屈表情 |
| 右键可可 | 打开菜单 | 喂食/装扮/设置/退出 |
| 鼠标在可可附近晃动 | 可可伸爪抓光标 | 爪子跟随鼠标轨迹 |

#### F4. 好感度与养成

```
好感度 0──100──300──600──1000
       │    │     │     │
       陌生  熟悉  亲密  家人
       │    │     │     │
       基础   解锁  解锁   解锁
       行为   新动画  装扮   彩蛋
```

- 摸头 +1、喂食 +5、在线时长每小时 +10
- 好感度解锁新行为和互动
- 好感度下降（长时间不互动，每天-2）

#### F5. 喂食系统

| 食物 | 效果 | 冷却 |
|------|------|------|
| 小鱼干 🐟 | 好感+5，兴奋10秒 | 5分钟 |
| 猫罐头 🥫 | 好感+15，满足睡觉 | 30分钟 |
| 猫薄荷 🌿 | 可可进入疯狂状态 | 1小时 |

### 3.2 增值功能（v1.1+）

#### F6. 装扮系统
- 帽子、围巾、眼镜等配饰
- 节日限定装扮（圣诞帽、新年发箍）
- 好感度解锁 + 内购

#### F7. 多猫模式
- 可可+娜娜+阿里同屏
- 猫咪之间互动（打闹、一起睡觉）
- Steam创意工坊支持自定义角色

#### F8. 桌面小工具
- 番茄钟：可可陪你专注，完成时庆祝
- 喝水提醒：可可叼着水杯出现
- 整点报时：可可举牌子

#### F9. 成就系统
- "初次见面"：首次摸头
- "猫奴认证"：累计摸头100次
- "小鱼干大亨"：喂食50次
- "熬夜冠军"：凌晨3点可可还在陪你

## 五、技术方案

### 5.1 技术选型

| 层 | 方案 | 理由 |
|----|------|------|
| 框架 | **Electron + HTML5 Canvas** | 跨平台、透明窗口成熟、动画性能好 |
| 备选 | Python + PyQt5 | 轻量但动画能力弱 |
| 动画 | CSS Sprite / Lottie | 帧动画流畅、资源体积小 |
| 存储 | SQLite（本地） | 好感度、设置、装扮数据 |
| 打包 | electron-builder | 一键打包 exe/msi |

### 5.2 动画资源规格

| 动画 | 帧数 | 尺寸 | 格式 |
|------|------|------|------|
| 待机眨眼 | 8帧 | 150×200 | PNG序列 |
| 走路 | 12帧 | 200×200 | PNG序列 |
| 爬窗 | 10帧 | 200×300 | PNG序列 |
| 睡觉 | 6帧（循环） | 200×150 | PNG序列 |
| 摸头反应 | 6帧 | 150×200 | PNG序列 |
| 吃鱼干 | 8帧 | 200×200 | PNG序列 |

### 5.3 窗口层级

```
z-index 最上层：可可猫窗口（alwaysOnTop）
         ↓
z-index 中层：右键菜单（popup）
         ↓
z-index 下层：Windows 桌面 / 其他应用
```

## 七、风险与依赖

| 风险 | 等级 | 应对 |
|------|------|------|
| 动画资源产出慢 | 高 | MVP先用静态图+简单位移，逐步替换帧动画 |
| 透明窗口兼容性 | 中 | 测试Win10/11，留降级方案（带背景色窗口） |
| Electron体积大 | 中 | 用electron-builder压缩，目标<80MB |
| 可可形象还原度 | 中 | 基于原图描边+Q版化，非写实路线 |
| 杀毒软件误报 | 低 | 代码签名证书 |
